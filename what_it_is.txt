# Helix OS

Helix OS is a local AI execution system built as a personal control plane
for a single machine. Natural language input is converted into a structured
JSON execution plan and run through a set of controlled, deterministic tools.

It is not a chatbot. Direct LLM conversation is a fallback, not the default.

---

## How it works

Three layers:

1. Interface — terminal input, streamed output, /commands
2. Control plane (core/) — routes input, builds execution plan, runs steps
3. Services — deterministic tools that perform the actual work

The planner (Ollama/gemma3:4b) reads user input and outputs a JSON plan:
  [{"tool": "filesystem", "action": "find_file", "args": {"name": "chat.py"}}]

Steps can reference earlier results via $stepId for multi-step chains.
Conversation history (last 10 messages) and persisted memories are injected
into the LLM context on every call.

---

## Capabilities

- File system: find, read, list, navigate, create, delete, move
- OS: CPU/RAM/swap/disk usage, process listing, kill by name or PID, shell execution
- Memory: persist facts to disk across sessions, injected into LLM context on every call
- Repository analysis: file walk, AST metadata extraction, LLM summarization
- Multi-step chaining: output of one step feeds into the next via $stepId
- Plan validation: every plan checked before execution, invalid plans fall back to chat
- Cross-platform: runs on Linux, Windows, Mac via platform adapter layer
- Conversation: LLM fallback when no tool fits

---

## Stack

- Python 3.10+
- Ollama (local, gemma3:4b) — no cloud APIs
- psutil for OS introspection
- requests for Ollama HTTP calls
- Config centralised in core/config.py

## Prerequisites

- ollama installed from ollama.ai
- model pulled: ollama pull gemma3:4b
- pip install -e .
- run: helix

---

## What it is not

This is a finished v1 CLI tool, not a platform.
No background tasks, no multi-agent workflows, no GUI, no plugin system.
