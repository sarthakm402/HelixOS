# AI Cyberdeck

AI Cyberdeck is a local-first AI execution system that functions as a personal control plane for interacting with files, memory, tools, and automated workflows on a local machine.

It is not a chatbot interface.

It is an execution-driven system where natural language is converted into structured plans and safely executed through controlled tools.

---

## Core Design

The system is built as a 3-layer execution stack:

### 1. Interface Layer (Frontend)
- Terminal UI and optional dashboards
- Accepts natural language commands
- Streams responses from execution engine

### 2. Control Plane (Core)
Responsible for decision-making and orchestration:

- Converts user input → structured execution plan
- Maintains short-term conversational context
- Manages tool selection and execution flow
- Handles multi-step workflows
- Maintains execution state across steps

This layer does NOT perform actions directly.

It only decides WHAT should happen.

---

### 3. Execution Layer (Services)

A set of deterministic tools that perform system actions:

- File system operations (search, read, navigate)
- Memory storage and retrieval
- Code analysis utilities
- LLM inference (generation, summarization)
- System interaction primitives

This layer does NOT decide.

It only executes.

---

## System Philosophy

AI Cyberdeck follows a strict separation of concerns:

- Language models interpret intent
- The control plane converts intent into structured plans
- Tools execute plans deterministically
- State is explicitly tracked across operations

No single component is allowed to act autonomously without structure.

---

## Capabilities

Current capabilities include:

- Natural language command execution
- File system navigation and search
- Memory persistence across sessions
- Repository analysis (AST + summarization)
- Multi-step tool execution via planning
- Streaming AI responses

---

## Future Direction

The system is designed to evolve into a local AI operating environment capable of:

- multi-agent workflows
- background task execution
- system-level automation
- structured long-term memory
- skill-based abstraction layers
- optional OS integration (controlled and sandboxed)

---

## Key Principle

AI Cyberdeck is not a conversational agent.

It is a **deterministic execution system controlled by language interfaces**.
